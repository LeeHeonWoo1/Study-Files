타입 안정성의 영향이 제일 크다. 이로 인해 코드에 버그가 많이 줄어든다.

타입 스크립트의 모든 자료형은 자바 스크립트와 동일하다. Number, Boolean, String, Array, Object 모두 동일하다. 차이점은, 에러 허용에 대한 폭이다.

예를 들어, [1, 2, 3, 4]라는 배열에 false라는 불리안 타입을 더하려고 한다면, 즉
[1, 2, 3, 4] + false의 결과는 => '1,2,3,4false'가 출력된다. 

또, 두 수를 인자로 받아 나눠주는 함수가 있다고 가정하자.

function devide(a, b){
  return a / b
}

이 함수의 경우 devide(2, 3)을 실행하면 0.66666666이라는 결과가 출력되지만, 
devide('xxxxxxx') 식으로 실행하면 당연히 에러가 나야한다. 받는 인자 수는 2개이며, 두 인자 모두 기본값이 설정되지 않았을 뿐더러 숫자형이 아니기에 에러가 나야하지만 NaN이라는 결과를 뱉을 뿐 에러를 표출하지 않는다.
